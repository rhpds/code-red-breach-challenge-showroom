== External Secrets

Openshift Secrets contain credentials and other sensitive information.

For security reasons, Secrets are hidden, not managed, and their content is stored in a Vault.

The mechanism of ExternalSecret synchronizes information from Vault and inject them into automatically created Openshift Secrets.


=== Vault

Vault is a key-value store.
The key is found under a specific path, similar to a folder path (i.e. /my/path/to/key).

Vault is divided into multiple Secret Store, which define the root path.

This adds security as the path to a key in an ExternalSecret is relative to what is defined in the Secret Store.


=== GitOps

External Secrets, like all the other components of the applications, follow the principle of GitOps.

They are synchronized from the declarative source files by ArgoCD.

The standard sources format is Helm Charts.

For the External Secrets, a variable in the Helm's values.yaml identify the Vault path to use.
