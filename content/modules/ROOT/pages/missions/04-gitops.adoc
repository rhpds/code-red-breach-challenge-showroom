== Openshift GitOps and ArgoCD

One of the main Automation Corporate standards is that all configuration follows a GitOps approach.

As a result, all managed Openshift Objects is created and maintain by ArgoCD.

Most of the teams use Helm chart as their source format, and a values.yaml file for parameters.

Those parameters should not be overriden.


== Operational tasks with GitOps and Pipelines

Contrary to Openshift DeploymentConfig object, Kubernetes Deployment objects don't react on an update of the underlying pod image.

To work around that issue, the Pipeline terminates with a 'resync' Task that restarts the pod, using 'oc delete pod ...'.

In the past, the Pipeline executed an 'argocd app sync' command, but the Task was replaced for security reason.
